## 1. 系统概述

### 1.1 三元协同教学理念

本系统构建了 **"教师 - 学生 - AI 智能体"** 三元协同教学结构，核心理念如下：

| 交互通道 | 描述 | 系统实现 |
|---|---|---|
| **学生 ↔ AI** | 高频互动，完成基础语言操练 | 学生端 `AISceneChat` 模块，基于 Gemini 2.5 Flash 模型 |
| **教师 ↔ AI** | 教师通过后台指令 AI 调整教学策略 | 情景任务发布、试卷生成、强化方案推送 |
| **教师 ↔ 数据** | 教师通过后台查看 AI 反馈的学情数据 | 教学仪表盘、学生详情页、作业批改系统 |

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────┐
│                    Docker Compose 编排                    │
│                                                         │
│  ┌─────────────┐   ┌──────────────┐   ┌──────────────┐ │
│  │  Frontend    │   │   Backend    │   │  PostgreSQL  │ │
│  │  Vite+React  │──▶│   FastAPI    │──▶│    16        │ │
│  │  Nginx:80    │   │   :8000      │   │    :5432     │ │
│  └─────────────┘   └──────┬───────┘   └──────────────┘ │
│                           │                             │
│                    ┌──────▼───────┐                     │
│                    │ Google AI    │                     │
│                    │ Gemini API   │                     │
│                    └──────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

- **前端**：Vite + React 18 + TailwindCSS，使用 `HashRouter` 进行路由管理
- **后端**：Python FastAPI，提供 RESTful API
- **数据库**：PostgreSQL 16，通过 SQLAlchemy ORM 操作
- **AI 引擎**：Google Gemini 2.5 Flash，作为德语助教对话核心
- **部署**：Docker Compose 三容器编排（前端 Nginx + 后端 uvicorn + 数据库）

### 1.3 前端 API 通信机制

系统使用统一的 Axios 实例（`src/api/request.js`）进行所有 API 请求：

- **Token 自动注入**：每次请求自动从 `localStorage` 读取 `authToken`，注入 `Authorization: Bearer <token>` 请求头
- **401 自动跳转**：若后端返回 401 状态码，前端自动清除 Token 并重定向至教师登录页
- **API 代理策略**：
  - 开发环境：Vite DevServer 将 `/api` 请求代理至 `http://localhost:8000`
  - 生产环境：Nginx 反向代理将 `/api` 请求转发至后端容器

## 2. 快速入门

### 2.1 环境部署 (Docker 一键启动)

```bash
# 1. 在项目根目录下配置 .env 文件
echo "GOOGLE_API_KEY=你的Gemini密钥" > .env

# 2. 一键启动所有服务
docker compose up -d

# 3. 访问系统
#    教师端入口：http://localhost/#/teacher/login
#    后端健康检查：http://localhost:8000/
```

> ⚠️ **注意**：首次启动时，Docker 会自动执行 `001_init_schema.sql` 建表和 `002_seed_demo.sql` 插入演示数据。

### 2.2 系统登录

- **访问地址**：

  | 环境 | 地址 |
  |---|---|
  | 本地开发 | `http://localhost:5173/#/teacher/login` |
  | Docker 部署 | `http://localhost/#/teacher/login` |

- **登录流程** (对应组件 `TeacherLogin.jsx`)：

  1. 输入 **工号/用户名** 和 **密码**
  2. 输入右侧显示的 **4位图形验证码**（字母数字组合，不区分大小写，点击图片可刷新）
  3. 点击 **"登录"** 按钮

- **后端认证逻辑**（`POST /api/auth/login`）：
  - 验证码在前端本地 Canvas 生成和校验，**不经过后端**
  - 登录成功后，后端返回 `mock-token`，前端将 Token 和用户信息存入 `localStorage`
  - 若输入的用户名在数据库中不存在，后端将**自动创建教师账号**（开发联调便利机制）

> ⚠️ **注意**：本系统当前使用 Mock Token 机制，暂不支持 JWT 验签。忘记密码请直接联系技术负责人重置数据库。

### 2.3 界面导航架构

登录成功后跳转至教学仪表盘，通过顶部操作按钮导航至各功能模块：

```
教师端路由结构 (HashRouter)
├── /teacher/login        → 教师登录页
├── /teacher/dashboard    → 教学仪表盘 (主页面)
├── /teacher/scenario     → 情景任务发布
├── /teacher/exam         → 智能试卷生成
└── /teacher/student/:id  → 学生详情页 (动态路由)
```

**教学仪表盘顶部导航按钮**：

| 按钮 | 功能 | 跳转路径 |
|---|---|---|
| 🔄 刷新 | 重新拉取仪表盘数据 | 不跳转（原地刷新） |
| ➕ 发布情景任务 | 进入情景配置页 | `/teacher/scenario` |
| 🎓 生成试卷 | 进入试卷生成页 | `/teacher/exam` |
| 🔴 退出 | 清除 Token 并登出 | 返回首页 `/` |

## 3. 功能模块详解

### 3.1 教学仪表盘 (Dashboard)

**组件**：`TeacherDashboard.jsx` | **API**：`GET /api/teacher/dashboard`

这是登录后的首页面，用于全局监控班级学情。

#### A. 核心指标卡片 (Stats Cards) — 4 维度

| 指标 | 字段名 | 数据来源 | 说明 |
|---|---|---|---|
| **班级总人数** | `stats.totalStudents` | 数据库 `Student` 表计数 | 当前班级已注册学生总数 |
| **人均互动时长** | `stats.avgDuration` | 后端聚合计算 | 单位：小时(h)，显示全班人均与 AI 对话总时长 |
| **平均综合得分** | `stats.avgScore` | `Student.overall_score` 取均值 | 全班综合能力评估分（0-100） |
| **任务完成率** | `stats.completionRate` | `Homework.status == "已完成"` 占比 | 所有已布置作业中标记为"已完成"的百分比 |

每张卡片右上角显示**趋势标签** (trend)：
- 以 `+` 或 `↑` 开头的趋势值显示为**绿色**（正向趋势）
- 其他值显示为**灰色**（中性趋势）

#### B. 学情监控列表 (Student Table)

仪表盘下方展示全班学生列表，包含以下信息：

| 列 | 说明 |
|---|---|
| **学生信息** | 姓名首字母头像 + 姓名 + 学号 |
| **活跃度** | 百分比进度条 + 数值（来自 `Student.active_score`） |
| **综合评分** | 色彩编码标签：≥90 绿色、≥80 蓝色、<80 橙色 |
| **薄弱环节** | 来自 `Student.weak_point`，如"虚拟式"、"被动语态" |
| **操作** | 箭头按钮，点击跳转至学生详情页 |

**功能操作**：

- **搜索过滤**：右上角搜索框支持按**姓名**或**学号**实时过滤学生列表
- **点击跳转**：点击表格行的任意位置均可跳转至该学生的详情页（`/teacher/student/:uid`），同时通过 React Router `state` 传递学生基本信息以实现"秒开"体验
- **数据刷新**：点击顶部"刷新"按钮重新调用 API，刷新成功/失败均有 Toast 通知

#### C. 离线降级机制

当 API 请求失败时，仪表盘会自动切换至**离线模式**：
- 页面顶部出现橙色提示条：`"网络请求失败，已切换至离线模式"`
- 使用内置的 `FALLBACK_DATA` 渲染基本界面框架，防止白屏

### 3.2 学生详情页 (Student Detail)

**组件**：`StudentDetail.jsx` | **API**：`GET /api/student/detail?id={uid}`

此页面是实施**个性化教学干预**的核心工具。

**路径**：从仪表盘学生列表 → 点击具体学生行

#### A. 个人信息卡片

展示学生的基本档案：
- **头像**：取姓名首字符，圆形靛蓝色背景
- **基本信息**：姓名、学号、班级
- **标签指标**：活跃度百分比（绿色标签）、综合评分（蓝色标签）

#### B. 德语能力模型 (Ability Bars)

系统以**水平进度条**形式展示学生的四项核心能力评分（0-100 分制）：

| 维度 | 德语名称 | 进度条颜色 | 数据来源 |
|---|---|---|---|
| 听力 | Hören | 🔵 蓝色 | `StudentAbility.listening` |
| 口语 | Sprechen | 🟠 橙色 | `StudentAbility.speaking` |
| 阅读 | Lesen | 🟢 绿色 | `StudentAbility.reading` |
| 写作 | Schreiben | 🟣 紫色 | `StudentAbility.writing` |

> **💡 技术说明**：能力值由后端 `StudentAbilityCRUD` 管理，当前演示数据中，听力 ≈ 活跃度+1，口语 ≈ 活跃度-2，阅读 ≈ 综合分，写作 ≈ 综合分-3。正式使用时应接入 AI 对话分析引擎。

#### C. AI 智能诊断卡片

位于能力模型右侧的**深色渐变卡片**（靛蓝→紫色），展示 AI 对该学生的个性化诊断文本。

- 诊断内容来自 `StudentAbility.ai_diagnosis` 字段
- 卡片底部有 **"一键推送强化方案"** 按钮

#### D. 推送个性化强化方案

**API**：`POST /api/student/push-scheme`

点击"一键推送强化方案"后，系统将执行以下操作：

1. 后端为该学生创建一个**个性化场景任务**（`Scenario` 记录，主题为"个性化强化方案"，难度为"自适应"，性格为"严谨纠错"）
2. 创建 `ScenarioPush` 记录，将方案推送状态标记为 `pushed`
3. 前端显示 Toast 通知推送结果

**请求载荷**：

```json
{
  "studentId": "2452001",
  "name": "李娜",
  "diagnosis": "李娜在虚拟式方面需要重点强化。",
  "timestamp": "2026-02-24T12:00:00.000Z"
}
```

#### E. 作业完成记录 (Homework Table)

展示该学生的所有作业列表，包含以下列：

| 列 | 说明 |
|---|---|
| **作业标题** | 图标区分类型（📝 文本 / 🎙️ 音频） |
| **提交日期** | 格式 `YYYY-MM-DD` |
| **状态** | 色彩编码标签：已完成(绿)、待订正(橙)、未提交(红)、进行中(蓝)、逾期补交(灰) |
| **得分** | 色彩编码：≥90 绿、≥80 蓝、≥60 橙、<60 红 |
| **操作** | "查看详情"链接，打开作业批改模态框 |

#### F. 作业批改模态框 (HomeworkModal)

点击"查看详情"后弹出全屏模态框，支持完整的作业审批流程：

**API 调用链**：

```
1. GET /api/homework/detail?id={homeworkId}  → 获取作业文件详情
2. POST /api/homework/save                    → 保存教师评分和反馈
```

**模态框包含三个区域**：

1. **作业文件下载**
   - 展示文件名、文件大小、上传时间
   - 点击"下载"触发浏览器原生下载（通过动态创建 `<a>` 标签实现）
   - 根据文件类型（`text` / `audio`）自动切换图标

2. **AI 智能评分**
   - 展示 AI 自动批改的评价文本（`aiComment` 字段）
   - 紫靛色渐变背景，带"自动批改完成"状态标签

3. **教师人工评分**
   - **得分输入**：数字输入框（0-100），带前端校验
   - **指导意见**：多行文本域，教师可输入手写反馈
   - **保存按钮**：调用 `POST /api/homework/save`，后端同步创建 `HomeworkReview` 记录并更新 `Homework` 表

### 3.3 情景任务发布 (Scenario Launch)

**组件**：`ScenarioLaunch.jsx` | **API**：`POST /api/scenario/publish`

此功能用于向全班发布统一的 AI 情景对话任务，控制学生端 `AISceneChat` 的行为。

#### 配置参数详解

**1. 情景主题 (Szenario)** — 卡片单选

| 主题 | 说明 |
|---|---|
| 慕尼黑机场问路 | 旅行场景：问路、指示牌、交通工具 |
| 餐厅点餐 (Bestellung) | 日常场景：菜单、点单、付款 |
| 校园迎新 (Campus) | 学术场景：报到、选课、与教授沟通 |

选中主题后卡片呈现紫色高亮 + ✅ 勾选图标。

**2. 难度等级 (Niveau)** — 下拉选择

| 等级 | 适用范围 |
|---|---|
| A1 - 入门 (基础词汇) | 使用 Top 500 常用词汇，简单句式 |
| A2 - 初级 (简单从句) | 包含从句结构，扩展词汇量 |
| B1 - 进阶 (流利交流) | 复杂句式，要求流利表达 |

**3. AI 角色性格** — 下拉选择

| 性格 | 教学行为 |
|---|---|
| 友好耐心 (Encouraging) | 鼓励表达，不纠正小错误，适合初学者建立信心 |
| 严谨纠错 (Strict) | 实时纠正每个语法错误，适合考前强化 |
| 幽默风趣 (Humorous) | 用轻松方式引导对话，提高学习趣味性 |

**4. 训练目标设定** — 复选框

| 条目 | 效果 |
|---|---|
| ☑ 强制使用完成时态 (Perfekt) | AI 在对话中要求学生必须使用现在完成时 |
| ☑ 包含至少 5 个 B1 级词汇 | AI 主动引入高级词汇并引导学生使用 |

#### 发布流程

点击 **"确认发布任务"** 后：

1. 前端构造配置载荷并发送至 `POST /api/scenario/publish`
2. 后端创建 `Scenario` 记录，生成唯一编号（格式：`SCN-YYYYMMDDHHmmss-xxxx`）
3. 后端遍历当前班级所有学生，为每人创建 `ScenarioPush` 记录（状态：`pushed`）
4. 前端收到成功响应后显示 Toast 通知（含任务 ID 和主题名称），1.5 秒后自动返回仪表盘

**请求载荷示例**：

```json
{
  "config": {
    "theme": "慕尼黑机场问路",
    "difficulty": "A1 - 入门 (基础词汇)",
    "persona": "友好耐心 (Encouraging)",
    "goals": {
      "requirePerfectTense": true,
      "requireB1Vocab": false
    }
  },
  "timestamp": "2026-02-24T12:00:00.000Z"
}
```

### 3.4 智能试卷生成 (Exam Generator)

**组件**：`ExamGenerator.jsx` | **API**：`POST /api/exam/generate`

基于学情数据的 AI 差异化出题系统。

#### A. AI 诊断建议区

页面顶部蓝色提示框展示 AI 基于班级学情分析的出题建议。当前系统预设关注：
- **被动语态** (Passiv)
- **虚拟式** (Konjunktiv)

> 💡 此区域的建议目前为静态文案，后续版本将接入 Gemini 实时生成。

#### B. 试卷结构配置

| 题型 | 参数名 | 范围 | 默认值 | 说明 |
|---|---|---|---|---|
| **语法填空 (Grammatik)** | `grammarItems` | 5-30 题 | 15 题 | 侧重变格 (Deklination) 与动词变位 (Konjugation) |
| **情景改写 (Schreiben)** | `writingItems` | 1-5 篇 | 2 篇 | 侧重句子重构能力 |

使用**滑块控件** (range input) 调节数量，实时显示当前值。

#### C. 分发策略

| 策略 | 参数值 | 说明 |
|---|---|---|
| **千人千面 (推荐)** | `personalized` | AI 根据每位学生的弱点自动替换 30% 的题目（默认选中，带 ⚡ AI 推荐标签） |
| **统一试卷** | `unified` | 全班使用完全相同的标准试卷，便于横向对比成绩 |

#### 生成流程

点击 **"开始生成试卷"** 后：

1. 前端发送配置至 `POST /api/exam/generate`
2. 后端创建 `Exam` 记录（编号格式：`EXM-YYYYMMDDHHmmss-xxxx`）
3. 后端遍历全班学生，为每人创建 `ExamAssignment`（状态：`assigned`）
4. 返回试卷 ID 和分发学生数，前端 Toast 通知成功后 1.5 秒返回仪表盘

## 5. 通知系统 (Toast)

系统使用自研 Toast 组件（`src/components/Toast.jsx`）替代浏览器原生 `alert()` 弹窗：

| 类型 | 调用方式 | 样式 | 自动消失 |
|---|---|---|---|
| ✅ 成功 | `toast.success('消息')` | 绿色背景 + 勾选图标 | 3.5 秒 |
| ❌ 错误 | `toast.error('消息')` | 红色背景 + 叉号图标 | 5 秒 |
| ℹ️ 提示 | `toast.info('消息')` | 蓝色背景 + 信息图标 | 3.5 秒 |

Toast 通知出现在屏幕**右上角**，支持多条同时堆叠显示，点击 × 可手动关闭。

## 6. 登出机制

教师端登出功能位于仪表盘页面**右上角的红色"退出"按钮**。

登出执行以下操作：
1. 清除 `localStorage` 中的 `authToken` 和 `userInfo`
2. 显示 Toast 提示 `"已安全退出"`
3. 500ms 后自动跳转回首页 (`/`)

> ⚠️ **安全说明**：如果在任意页面的 API 请求中收到 `401` 响应，统一 Axios 拦截器会自动清除凭证并跳转至教师登录页 (`/teacher/login`)。

## 7. 部署配置

### 7.1 Docker Compose 容器编排

系统通过 `docker-compose.yml` 编排为三个容器：

| 容器 | 镜像 | 端口映射 | 职责 |
|---|---|---|---|
| `german-db` | `postgres:16` | `5432:5432` | PostgreSQL 数据库，数据通过 `pg_data` 卷持久化 |
| `german-backend` | 自定义 (Dockerfile) | `8000:8000` | FastAPI 后端，通过 `DATABASE_URL` 连接数据库 |
| `german-frontend` | 自定义 (Dockerfile) | `80:80` | Nginx 提供静态资源 + 反向代理 `/api` |

### 7.2 环境变量

| 变量 | 所在容器 | 说明 |
|---|---|---|
| `GOOGLE_API_KEY` | backend | Google Gemini API 密钥（从项目根目录 `.env` 读取） |
| `DATABASE_URL` | backend | PostgreSQL 连接串，Docker 中设为 `postgresql+psycopg://postgres:postgres@db:5432/sitp_german_ai_agent` |

## 8. 故障排除

| 现象 | 可能原因 | 解决方案 |
|---|---|---|
| **登录页验证码不显示** | 浏览器禁用了 Canvas 或 JavaScript | 检查浏览器设置，确保未使用无头模式 |
| **登录后跳转白屏** | 后端服务未启动或网络不通 | 检查 `docker compose ps` 确认所有容器运行正常 |
| **仪表盘显示"离线模式"** | 后端 API 请求失败 | 1. 检查后端日志 `docker logs german-backend`；2. 尝试 `Ctrl+Shift+R` 强制刷新 |
| **生成试卷一直 loading** | Google Gemini API 响应超时 | 1. 检查 `.env` 中 API Key 是否有效；2. 检查服务器网络连通性 |
| **无法发布情景任务** | 后端数据库连接异常 | 检查 `docker logs german-db`，确认 PostgreSQL 正常运行 |
| **作业下载无响应** | 后端未返回有效文件 URL | 检查 `HomeworkCRUD` 中 `file_url` 字段是否为有效链接 |
| **Toast 通知不显示** | App 未被 `<ToastProvider>` 包裹 | 确认 `main.jsx` 中 App 组件外层有 `<ToastProvider>` |
| **页面样式异常** | TailwindCSS 未正确编译 | 执行 `npm run build` 检查构建输出是否有错误 |
