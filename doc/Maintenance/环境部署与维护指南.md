# SITP German AI Agent 环境部署与维护指南

> **部署方式**: Docker Compose (全容器化)
> **操作系统**: 兼容 Linux (Ubuntu/CentOS), Windows (Docker Desktop), macOS

---

## 1. 硬件与环境要求

- **CPU**: 2核 及以上
- **内存**: 4GB 及以上
- **磁盘**: 20GB 可用空间
- **软件**: Docker 24+ & Docker Compose 2+
- **网络**: 需能访问 `api.generativelanguage.google.com` (可能需要代理)

---

## 2. 快速部署步骤

### 2.1 获取源码与配置
1. 下载项目源码至服务器。
2. 在根目录创建 `.env` 文件：
   ```bash
   GOOGLE_API_KEY=你的_GEMINI_API_KEY
   # 如果需要代理
   HTTP_PROXY=http://your-proxy-ip:port
   HTTPS_PROXY=http://your-proxy-ip:port
   ```

### 2.2 启动服务
在项目根目录执行：
```bash
docker compose up -d --build
```

### 2.3 验证状态
```bash
docker compose ps
# 预期应有 3 个容器处于 Up 状态
```
- 前端：`http://localhost`
- 后端接口：`http://localhost:8000/docs`
- 数据库：`localhost:5432`

---

## 3. 常见维护操作

### 3.1 查看日志 (Troubleshooting)
```bash
# 查看所有容器日志
docker compose logs -f

# 仅查看后端报错
docker compose logs german-backend -f --tail 100
```

### 3.2 数据库备份与恢复
**手动备份**:
```bash
docker exec -t german-db pg_dumpall -c -U postgres > backup_$(date +%Y%m%d).sql
```

**恢复数据**:
```bash
cat backup_file.sql | docker exec -i german-db psql -U postgres
```

### 3.3 重置演示数据
若需清除所有数据并恢复至 Demo 状态：
```bash
docker compose down -v  # 注意：这会删除 pg_data 卷中的所有数据
docker compose up -d
```

## 4. 生产环境安全建议

1. **禁用自动创号**: 在 `main.py` 的 `/api/auth/login` 中，注释掉 `UserCRUD.create` 逻辑，改为手动录入教师。
2. **更换默认密码**: 修改 `docker-compose.yml` 中的 `POSTGRES_PASSWORD`。
3. **启用 HTTPS**: 建议在 `german-frontend` (Nginx) 前置一个负载均衡器（如阿里云 SLB 或域名 Nginx）并配置 SSL 证书。
4. **API 密钥保护**: 定期更换 `GOOGLE_API_KEY`，并避免将包含 Key 的 `.env` 文件提交至版本控制系统。

---

## 5. 版本更新指南

1. 拉取最新代码：`git pull origin main`
2. 重建并启动：`docker compose up -d --build`
3. 检查数据库变更：若有 SQL 变更，需通过 `docker exec` 手动运行补丁脚本或使用 Alembic (如后续集成)。
