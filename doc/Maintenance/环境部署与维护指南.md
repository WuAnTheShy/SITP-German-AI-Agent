# SITP German AI Agent 环境部署与维护指南

> 部署方式：Docker Compose（前端 Nginx + 后端 FastAPI + PostgreSQL）  
> 适用系统：Linux / Windows（Docker Desktop）/ macOS

---

## 1. 部署前检查

### 1.1 资源建议

- CPU：2 核及以上
- 内存：4 GB 及以上（推荐 8 GB）
- 磁盘：20 GB 及以上可用空间

### 1.2 软件版本

- Docker 24+
- Docker Compose 2+

### 1.3 网络要求

- 若使用 Qwen 在线模式，服务器需能访问 DashScope API。
- 若使用 LM Studio 本地模式，后端容器需可访问宿主机 1234 端口。

---

## 2. 环境变量基线

根目录必须存在 .env 文件，至少包含以下关键项：

```dotenv
APP_ENV=production

POSTGRES_USER=postgres
POSTGRES_PASSWORD=请设置高强度数据库密码
POSTGRES_DB=sitp_german_ai_agent

INIT_ADMIN_USERNAME=admin
INIT_ADMIN_PASSWORD=请设置高强度管理员密码

AUTH_TOKEN_SECRET=请设置32位以上随机密钥
AUTH_TOKEN_TTL_HOURS=12
ALLOW_LEGACY_TOKENS=false

ENABLE_DEMO_SEED=false
RESET_ADMIN_PASSWORD_ON_STARTUP=false
RESET_DEMO_PASSWORDS_ON_STARTUP=false

LLM_PROVIDER=qwen
QWEN_API_KEY=请填写你的Qwen密钥

DEBUG_LLM_LOGS=false
```

说明：

- 同域部署时 CORS_ALLOW_ORIGINS 可留空。
- 切换到 LM Studio 时，需要把 LLM_PROVIDER 改为 lmstudio，并设置 LMSTUDIO_BASE_URL 与 LMSTUDIO_MODEL。

## 3. 首次部署（无历史数据卷）

适用条件：

- 机器首次部署本项目
- 或已执行过 docker compose down -v 清空数据卷

执行命令：

```bash
docker compose up -d --build
docker compose ps
docker compose logs -f backend
```

成功判定：

- docker compose ps 显示 frontend、backend、db 均为 Up
- backend 日志出现 Application startup complete
- 浏览器访问 http://localhost（或服务器 IP）可看到登录页

---

## 4. 升级部署（已有历史数据卷）

适用条件：

- 已在线运行过本项目
- 需要更新代码、镜像、配置或模型参数

执行命令：

```bash
git pull
docker compose up -d --build
docker compose ps
docker compose logs -f backend
```

### 4.1 数据库密码同步（高频故障）

现象：

- backend 日志出现 password authentication failed for user "postgres"
- 前端请求 /api 接口出现 502

原因：

- PostgreSQL 首次初始化后，库内密码不会随 .env 自动变化。

处理：

```bash
docker exec -u postgres german-db psql -d sitp_german_ai_agent -c "ALTER USER postgres WITH PASSWORD '你的新密码';"
docker compose restart backend
docker compose logs -f backend
```

## 5. 启动后验收清单

```bash
# 1) 服务状态
docker compose ps

# 2) 后端健康检查
curl -i http://localhost:9000/

# 3) 反向代理链路检查（返回 401/200 视为链路通，502 视为异常）
curl -X POST http://localhost/api/auth/login \
   -H "Content-Type: application/json" \
   -d '{"username":"admin","password":"wrong_password"}'
```

验收标准：

- 无 502
- 后端日志无持续数据库重连异常
- 管理员账号可以正常登录

## 6. 常用维护命令

```bash
# 查看状态
docker compose ps

# 查看所有日志
docker compose logs -f

# 分服务看日志
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f db

# 重启服务
docker compose restart

# 停止并移除容器（保留卷）
docker compose down

# 停止并移除容器+卷（会删除数据库）
docker compose down -v
```

## 7. 备份与恢复

### 7.1 手动备份

```bash
docker exec -t german-db pg_dump -U postgres -d sitp_german_ai_agent > backup_$(date +%Y%m%d_%H%M%S).sql
```

### 7.2 手动恢复

```bash
cat backup_xxx.sql | docker exec -i german-db psql -U postgres -d sitp_german_ai_agent
```

恢复后建议执行：

```bash
docker compose restart backend
docker compose logs --tail 100 backend
```

## 8. 常见问题排查

| 现象 | 典型原因 | 建议处理 |
| --- | --- | --- |
| required variable POSTGRES_PASSWORD is missing | .env 缺少必填项 | 补齐后重启 docker compose |
| password authentication failed for user "postgres" | .env 与库内密码不一致 | 执行 ALTER USER 同步密码 |
| 前端 502 Bad Gateway | backend 未就绪或启动失败 | 先看 backend 日志定位 |
| AI 无回复 | LLM_PROVIDER 或 Key 配置错误 | 核对 .env 与后端日志的 Provider/Model |

---

## 9. 生产安全建议

1. 不要把 .env 提交到仓库。
2. 定期轮换 QWEN_API_KEY、AUTH_TOKEN_SECRET、数据库密码。
3. 对外只暴露前端入口，后端与数据库端口尽量仅内网可见。
4. 在网关或反向代理层配置 HTTPS。
5. 升级前先备份数据库，再进行镜像更新。
