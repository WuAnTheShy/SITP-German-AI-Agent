# SITP German AI Agent 智能体集成设计说明

> 核心引擎：Qwen（在线）或 LM Studio（本地）  
> 模型角色：教师教研助手 + 学生德语助教

---

## 1. 集成架构

```text
[前端 React] --> [后端 FastAPI] --> [Qwen API 或 LM Studio]
                      |                    |
                      +-----> [PostgreSQL]-+
```

说明：

- 前端不直接调用模型，所有模型请求统一经后端服务。
- 后端按 LLM_PROVIDER 环境变量切换模型通道。
- 对话与业务数据持久化在 PostgreSQL 中，支撑多轮上下文和结果追溯。

---

## 2. Provider 运行模式

### 2.1 Qwen 在线模式

- 环境变量：LLM_PROVIDER=qwen
- 关键配置：QWEN_API_KEY
- 可选配置：QWEN_API_URL、LLM_MODEL
- 特点：部署简单，适合生产环境。

### 2.2 LM Studio 本地模式

- 环境变量：LLM_PROVIDER=lmstudio
- 关键配置：LMSTUDIO_BASE_URL
- 模型配置：优先使用 LLM_MODEL，其次 LMSTUDIO_MODEL
- 可选配置：LMSTUDIO_API_KEY
- 特点：离线可用，适合本地开发与内网场景。

Docker 场景中 LMSTUDIO_BASE_URL 建议使用 http://host.docker.internal:1234。

---

## 3. 角色提示词策略

### 3.1 教师教研助手

- 输出语言：中文为主。
- 任务目标：学情分析、课堂建议、作业与试卷设计辅助。
- 风格要求：结构化、可执行、可解释。

### 3.2 学生德语助教

- 输出语言：默认德语 + 必要中文解释。
- 任务目标：语法纠错、表达优化、学习建议。
- 风格要求：鼓励性反馈，兼顾准确与可理解性。

---

## 4. 关键实现点

### 4.1 结构化输出

- 试卷生成等场景要求模型返回结构化 JSON。
- 后端对模型返回做清洗与解析，解析失败时走兜底策略。

### 4.2 会话记忆

- 普通对话通过会话表和消息表维持上下文。
- 学生情景对话支持状态查询、清空重开、历史延续。
- 学生与教师对话在固定轮数后刷新长期摘要（long_memory_summary）。

### 4.3 稳定性与安全

- 模型调用配置可通过环境变量热切换并重启生效。
- 生产环境关闭调试日志，避免输出敏感信息。
- 对网络异常、模型返回异常做统一失败兜底。

---

## 5. 常见问题

1. 对话无回复：优先检查 LLM_PROVIDER 与对应密钥/地址是否匹配。
2. Docker 下 LM Studio 不通：检查是否使用 host.docker.internal。
3. 结构化解析失败：检查提示词约束与模型响应格式，必要时启用兜底题库。
