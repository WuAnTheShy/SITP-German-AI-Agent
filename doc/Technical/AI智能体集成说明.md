# SITP German AI Agent 智能体集成设计说明

> **核心引擎**: Google Gemini 2.5 Flash
> **SDK 版本**: 新版 `google-genai` (Python)
> **模型角色**: 教师教研助手 & 学生德语助教

---

## 1. 架构示意

```
[前端 (React)] --(消息/配置)--> [后端 (FastAPI)] --(Prompt + Context)--> [Gemini API]
                                     ^                                     |
                                     |                                     v
                                [数据库 (PG)] <-------(结构化响应解析)--------+
```

系统通过**系统提示词 (System Instruction)** 和**上下文注入 (Context Injection)** 实现角色的专业化分工。

---

## 2. 角色定义与系统提示词 (System Prompts)

### 2.1 教师教研助手 (Teacher Assistant)
- **目标**: 协助教师分析学情记录、制定教案、生成高质量试题。
- **系统提示词摘要**:
  > "你是一个高级教学AI教研助手，负责协助德语教师分析学情、制定教案和出题等。请用中文与教师进入沟通，提供专业、基于数据的教学建议。"
- **应用场景**: 教学仪表盘侧边对话框、推送方案建议、智能出题。

### 2.2 学生德语导师 (Student Tutor)
- **目标**: 提供全天候口语/文字练习、语法纠错、辅助写作和单词学习。
- **系统提示词摘要**:
  > "你是同济大学的 AI 德语助教，帮助学生学习德语。回复简洁准确。除非用户要求中文，否则默认用德语回答并在括号内给出中文翻译。"
- **应用场景**: 任务中心对话、自由提问、写作批改、单词例句生成。

## 3. 核心集成技术点

### 3.1 跨语言交互 (Cross-Language Consistency)
- AI 始终被指令在德语教学语境下运行，系统通过 Prompt 确保其具备**即时翻译**和**语法诊断**能力。

### 3.2 结构化数据解析 (JSON Structured Output)
- 在智能试卷生成（`ExamGenerator`）中，后端通过复杂的 Prompt 强制 AI 输出严格符合 JSON 格式的题目数据。
- 后端使用正则表达式或 `json.loads` 对响应进行预处理（去除 Markdown 代码块），随后存入数据库 `exams` 表的 `content` 字段。

### 3.3 多轮对话记忆管理 (Memory Management)
- **简易内存管理**: 后端使用全局列表 `_teacher_chat_history` 和 `_student_chat_history` 维护最近 20 轮对话。
- **持久化会话**: 针对学生情景任务，使用 `chat_sessions` 和 `chat_messages` 表存储历史，每次请求时动态构建 `History` 列表发送给 Gemini。

### 3.4 代理与超时配置
- **代理支持**: 后端自动读取 `.env` 中的 `HTTP_PROXY`，确保国内环境下与 Google API 的连通性。
- **大模型超时**: 针对资源密集的出题请求，`http_options` 超时显式设置为 `90,000ms` (90秒)。

## 4. Prompt 工程案例

### 试卷生成 Prompt 策略 (Few-Shot Pattern)
- **输入**: 题型数量、难度 B1、重点考察某语法点。
- **要求**: 
  1. 题目必须不同且具有代表性。
  2. 选项需要具备迷惑性。
  3. 必须输出特定 JSON 格式。
- **容错处理**: 如果 AI 生成失败或格式错误，系统内建了多达 8 个语法考点的 **Fallback 题库** 作为兜底。
