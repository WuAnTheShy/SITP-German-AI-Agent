# SITP German AI Agent 系统流程与架构

本文通过流程图和组件图解释系统在教学业务、AI 调用、部署运维三个维度的工作方式。

---

## 1. 总体架构

```mermaid
graph LR
    subgraph Browser
        U[用户浏览器]
    end

    subgraph Deploy[Docker Compose]
        FE[Nginx + React SPA]
        BE[FastAPI + Uvicorn]
        DB[(PostgreSQL 16)]
    end

    subgraph LLM[LLM Provider]
        QW[Qwen API]
        LM[LM Studio]
    end

    U --> FE
    FE --> BE
    BE --> DB
    BE --> QW
    BE --> LM
```

说明：

- 外部流量统一进入前端 Nginx，再转发至后端 /api。
- 后端通过环境变量切换 Qwen 或 LM Studio。
- 数据库持久化在 Docker 卷中，升级部署时默认保留。

## 2. 认证与鉴权流程

```mermaid
sequenceDiagram
    participant C as 客户端
    participant A as Auth Router
    participant D as PostgreSQL

    C->>A: POST /api/auth/login 或 /api/auth/student-login
    A->>D: 查询用户并校验密码哈希
    D-->>A: 用户信息
    A-->>C: 返回 access_token + role
    C->>A: 后续请求携带 Authorization: Bearer token
    A->>A: 按角色执行 require_admin/teacher/student
    A-->>C: 返回业务数据或 403
```

## 3. 教学主流程

### 3.1 教师发布任务与学生作答

```mermaid
flowchart TD
    A[教师发布情景任务/试卷] --> B[后端写入任务与分发记录]
    B --> C[学生端任务中心加载待办]
    C --> D[学生完成作答并提交]
    D --> E[后端自动评分/入库]
    E --> F[教师端查看结果并批改]
    F --> G[更新学习统计与反馈]
```

### 3.2 AI 对话流程（教师/学生通用）

```mermaid
sequenceDiagram
    participant C as 客户端
    participant B as Backend
    participant L as Qwen 或 LM Studio
    participant D as DB

    C->>B: 发送消息（可携带 session_id）
    B->>D: 读取历史消息
    B->>B: 组装系统提示词与上下文
    B->>L: 请求模型生成回复
    L-->>B: 返回文本或结构化结果
    B->>D: 持久化会话消息
    B-->>C: 返回 AI 回复
```

## 5. 部署流程分支

```mermaid
flowchart TD
    S[开始部署] --> J{是否已有历史数据库卷}
    J -- 否 --> F[首次部署: docker compose up -d --build]
    J -- 是 --> U[升级部署: docker compose up -d --build]
    F --> V[验证: ps/logs/登录]
    U --> P{出现 postgres 密码认证失败?}
    P -- 否 --> V
    P -- 是 --> A[执行 ALTER USER 同步库内密码]
    A --> R[重启 backend]
    R --> V
```

## 6. 故障定位路径

```mermaid
flowchart LR
    X[前端报错或登录失败] --> Y{HTTP 状态码}
    Y -- 502 --> Z[检查 backend 容器与日志]
    Z --> Z1[重点排查数据库连接和密码]
    Y -- 401 --> W[鉴权失败: 检查账号密码/token]
    Y -- 403 --> W1[角色权限不足]
    Y -- 500 --> Q[查看 backend 堆栈与路由逻辑]
    Z1 --> K[修复后重试登录和核心接口]
    W --> K
    W1 --> K
    Q --> K
```

## 7. 架构演进建议

1. 把高频业务补齐自动化测试用例，降低回归风险。
2. 引入结构化日志与追踪 ID，缩短线上故障定位时间。
3. 逐步收敛数据库启动补丁到迁移脚本，降低运行时复杂度。
4. 为 AI 调用增加限流、超时与降级策略，提升高并发稳定性。
